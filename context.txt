FoundrAI 2.0 — Full PRD + Project 
Context 
What Is FoundrAI 2.0 
FoundrAI 2.0 is an autonomous multi-agent business validation engine. A founder describes 
their startup idea — either via a plain text prompt or a structured intake form — and a team 
of 5 specialized AI agents debate the strategy, score it through a consensus protocol, and 
produce a complete validated business plan with real market data, GitHub deployment, and 
PDF export. 
The system is not a chatbot. It is a deliberation engine: agents critique each other, revise 
proposals through negotiation rounds, and only converge when a consensus score of 7.5/10 
is reached. Every output is grounded in live external data — not hallucinated. 
Current Stack 
Backend: FastAPI + Uvicorn, Python 3.11, HuggingFace Inference (Qwen2.5), LangChain 
@tool, ChromaDB RAG, Docker 
Frontend: Next.js 16, Tailwind CSS, Zustand, SVG-based pentagon arena, SSE streaming 
External APIs: Tavily (web search), GitHub REST + PyGithub (deployment), PyTrends 
(Google Trends), Reddit PRAW (community data), HuggingFace Inference API

Full PRD 
1. Product Overview 
Name: FoundrAI 2.0 Type: Autonomous Multi-Agent Business Validation Engine Target 
User: Early-stage founders, hackathon participants, solo entrepreneurs, product managers 
validating new ideas Core Promise: Enter your idea. 6 AI agents debate it with real market 
data. Get a validated business plan, legal check, GitHub repo, and PDF — in under 3 
minutes. 
2. Input System — Dual Mode 
Mode A: Free Text Prompt 
User types their idea in plain English in the terminal-style textarea. Same as current. 
Suitable for technical users who know how to write a good prompt. 
Mode B: Structured Intake Form 
User fills an 11-field form. The system auto-generates a rich structured prompt from their 
answers and passes it to the agent pipeline. Suitable for non-technical founders.

3. Agent Upgrades 
3.1 CEO Agent — Upgraded 
New outputs: 
● Workforce plan: suggested founding team roles with headcount per phase (Idea / 
MVP / Scale) 
● Timeline: milestone roadmap (Month 1-3, 3-6, 6-12, 12-24) with specific deliverables 
● Estimated cost breakdown: real cost estimates per phase using market rate data (not 
made up) 
Tool added: get_salary_benchmarks(role, location) — queries public salary 
APIs or uses RAG data to estimate team cost 
Output format: Structured markdown with a timeline table and a cost table alongside the 
prose proposal 
3.2 Marketing Agent (CMO) — Upgraded 
New outputs: 
● Influencer suggestions: real named influencers in the relevant niche with follower 
count, platform, engagement rate — sourced from Tavily search for "{niche} top 
influencers 2025" 
● Channel breakdown: primary + secondary channels with estimated CAC per channel 
● Differentiation statement: one crisp sentence vs named competitors 
Tool added: search_influencers(niche) — Tavily search returning real names and 
handles 
3.3 Finance Agent (CFO) — Upgraded 
New outputs: 
● ROI projection: Year 1, Year 2, Year 3 with assumptions stated 
● Burn rate estimate: monthly burn at each stage 
● Break-even analysis: month number and revenue required 
● Return on Investment table: investor perspective with realistic multiples 
● Feasibility score: explicit 1-10 with reasoning 
Frontend change: CFO tab on results page shows charts — line chart for revenue 
projection, bar chart for burn rate, single ROI number highlighted. Uses Recharts library 
already available. 
Output format: Structured sections with explicit numeric tables the frontend can parse into 
chart data 
3.4 Developer Agent — Upgraded 
New outputs: 
● Detailed README: not initial code, just a comprehensive README with architecture 
diagram in ASCII, setup steps, API reference table, environment variables table 
● Docker Compose: production-grade with health checks, named volumes, 
environment-specific configs, comments explaining each service 
● No starter code generated — README and Docker Compose only, done well 
Prompt change: Remove instruction to generate app/main.py. Focus LLM tokens on 
making README and docker-compose excellent and specific to the idea. 
3.5 Risk Agent — Upgraded 
New outputs: 
● Real-time Reddit posts: actual post titles and snippets surfaced from 
search_reddit_pain_points shown as cards in the frontend live feed — not just 
summarized 
● Each Reddit result shown as a card: subreddit name, post title, upvote range, 2-line 
snippet, relevance tag 
Frontend change: New RedditFeed.jsx component in the active stage right column — 
shows Reddit cards as they arrive via SSE, alongside the existing live feed 
SSE event added: reddit_result — { type: "reddit_result", posts: 
[{subreddit, title, body_snippet, score_range}] } 
3.6 Legal Agent — New 
Purpose: Checks the startup idea for legal exposure before the founder commits resources. 
What it checks: 
● Patent landscape: are there existing patents that could block this product 
● Regulatory compliance: industry-specific regulations (GDPR, HIPAA, FINRA, FDA 
depending on data_sensitivity field) 
● Country-specific policy: primary market legal requirements (incorporaton, licensing) 
● Labour law: employment vs contractor classification risk, equity/vesting compliance 
● IP ownership: open source license conflicts in the proposed tech stack 
Tools: 
● search_patents(idea) — Google Patents via Tavily search 
● search_regulations(industry, country) — Tavily search for regulatory 
requirements 
● query_legal_kb(topic) — ChromaDB over legal_frameworks.txt and 
labour_laws.txt 
Output format: 
Legal Risk Score: X/10 (lower = more risk) 
| Area | Finding | Severity | Recommendation | 
|---|---|---|---| 
| Patents | 3 relevant patents found | Medium | Design around claim 4 | 
| GDPR | User data stored → consent required | High | Add consent flow | 
| Labour | Contractors >6mo = employee risk | Medium | Use proper agreements | 
Frontend: Legal tab added to results page. LegalReport.jsx renders the structured table 
with color-coded severity badges. Legal agent runs in parallel with Phase 2 critiques. 
Phase change: Phase 2 is now "Parallel Critique + Legal Scan" — Legal runs concurrently 
with Dev/Finance/Marketing/Risk.

Consensus is still computed from CEO/Dev/Finance/Marketing/Risk (5 agents, threshold 
7.5/10). Legal risk score is shown separately as a standalone metric.
